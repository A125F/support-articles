If your build is taking more than 1 hour you will see that your build has stopped and you are on the free tier you will notice that if you view your jobs there will be a job called: Build timed out.

An example of this is shown below:

>Build timed out

Upgrade your pricing plan to take advantage of longer build times
context deadline exceeded

The reason for this change is the new pricing plan has been put in place which limits builds on the free plan to be able to build for a maximum of 1 hour before a timeout will occur and stop the build.

As you can see from the output of your job the build has a recommendation to upgrade your plan as this will allow you to continue to build with longer build times.

If you would like to continue using the free plan, we would recommend switching to the large resource class, available on the free plan.

It will give you access to more resources which will help reduce the build duration, thanks to additional CPU and memory.

More information about resource classes can be found here: https://circleci.com/docs/2.0/executor-types/#available-docker-resource-classes

To upgrade your build to use the large resource class you can add: resource_class: large an example of this can be seen below:

Current:

    docker:
      - image: ubuntu:bionic

Proposed:

    docker:
      - image: ubuntu:bionic
    resource_class: large

Additional information about our plans and available classes for each  plan they can be found here

In addition to utilizing a larger resource class size, another option is to utilize parallelism. This is especially helpful if the build encountering this issue is running tests.

We have information here on utilizing the feature: https://circleci.com/docs/2.0/parallelism-faster-jobs/

We have some additional details at the following blog post: https://circleci.com/blog/six-optimization-tips-for-your-config/

As long as your test suite allows you to select a specific set of tests to run, you should be able to execute them in parallel.

An example of implementing this can be found below:

Current:

    docker:
      - image: ubuntu:bionic

Proposed:

    docker:
      - image: ubuntu:bionic
    parallelism: 4

If you are still facing timeout issues after implementing the above suggestion, please raise a support ticket with the CircleCI support team.
